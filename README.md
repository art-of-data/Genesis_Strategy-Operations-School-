This project analyzes key product and marketing metrics of a mobile application over 2025.

The goal is to evaluate:
- User growth and retention
- Monetization efficiency
- Marketing performance (CAC vs LTV)

Dataset includes monthly data on downloads, MAU, revenue, and user behavior.

# Metrics Description

- Downloads — number of new users acquired
- MAU — monthly active users
- MRR — monthly recurring revenue
- ARPU — average revenue per user
- ARPPU — average revenue per paying user
- Conversion Rate — % of paying users
- Churn Rate — % of users lost monthly
- LTV — customer lifetime value
- CAC — customer acquisition cost
- LTV/CAC Ratio — profitability indicator

# Calculations

ARPU = MRR / MAU

ARPPU = MRR / Paying Users

Conversion Rate = Paying Users / MAU

Revenue per Download = MRR / Downloads

LTV = ARPU / Churn Rate

CAC = Avg CPI

LTV/CAC Ratio = LTV / CAC

# Analysis & Insights

1. User Base Decline
MAU decreased from 180,000 in January to 105,000 in December (-42%), indicating weak retention.

2. Revenue Decrease
MRR dropped from $86.4K to $60K (-30%), following the decline in active users.

3. Monetization Improvement
- ARPU increased from $0.48 to $0.57
- Conversion rate improved from 10% to 11.9%

This suggests better monetization of remaining users.

4. Rising Acquisition Costs
CAC increased from $1.8 to $2.6 (+44%), making user acquisition more expensive.

5. LTV Decline
LTV decreased from ~173 to ~77 due to increasing churn.

6. Profitability Drop
LTV/CAC ratio declined from 90 to below 30, indicating worsening unit economics.

# Conclusion

Despite improvements in monetization metrics, the overall business performance declined due to:

- Increasing churn
- Rising acquisition costs
- Shrinking user base

The product is becoming less sustainable over time.

Key focus areas:
- Improve retention
- Optimize acquisition channels
- Reduce churn
